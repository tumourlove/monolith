## Release Blockers Found

1. **FM Row 14:** `owner_resolver=banana` is silently accepted and coerced to `owning_player_pawn` (default). Spec mandates rejection with "Unknown owner_resolver" + valid list.
2. **FM Row 15:** `format=format_string` with template missing `{0}` slot is silently accepted and persisted. Spec mandates rejection.

Both are validation-gap bugs in `MonolithGASUIBindingActions.cpp` (`ParseOwner` falls through to default; `ParseFormat` does no slot-presence check on FormatString). Recommend a follow-up plan to add the missing validations.

## Tooling Constraint Surfaced

`ui_query` MCP dispatcher is NOT loadable through ToolSearch despite the `ui` namespace registering 130+ actions per `monolith_discover`. This means:
- `ui::bind_widget_to_attribute` and the 3 sister aliases were unreachable through agent tooling — only `gas::*` canonical names exercised.
- `ui::create_widget_blueprint` not reachable; agent had to duplicate existing UWidgetBlueprints from elsewhere in the project as fixture sources.
- This is environmental (MCP wrapper) not a Monolith bug, but blocks the ui-namespace alias from being directly tested by an agent.

## Other Findings

- **No `LogMonolithGAS` lines emitted by the bind/unbind/list/clear handlers.** Only the module-load message `MonolithGAS: Loaded (134 actions, GBA=available)` appears. Spec TC1.1 expects per-binding-fire trace lines and TC1.7 expects grace-period log capture; neither will be possible without adding logging. Recommend: add `UE_LOG(LogMonolithGAS, Verbose, ...)` per binding fire in `MonolithGASAttributeBindingClassExtension::ApplyValue`.
- **WBP compile post-modification:** Both `WBP_BindTest_ProgressBar` and `WBP_BindTest_Image` returned `status: UpToDate, errors: 0, warnings: 0` after multiple bind/unbind/clear cycles. The compile pipeline is sound.
- **Spec doc has a wrong prerequisite:** §Setup #4 claims "Leviathan ships `ULeviathanVitalsSet`". It does not. The fallback (disposable `AS_TestVitals` BP) is the only path. Update the spec to either ship the C++ AttributeSet or remove the false claim.
